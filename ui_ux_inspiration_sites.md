## How to Use This Directory (The 3-Step Research Method)

To build premium interfaces, don't just copy screenshots. Follow this industry-standard design workflow when doing visual research:

```
┌────────────────────────┐      ┌────────────────────────┐      ┌────────────────────────┐
│  1. BEHAVIOR & FLOWS   │ ───> │   2. LAYOUT & SYSTEM   │ ───> │  3. POLISH & VISUALS   │
│  How does user move?   │      │ How are elements set?  │      │ Colors, text, hover fx │
│  (Mobbin, Page Flows)  │      │  (SaaSUI, Screenlane)  │      │ (Godly, Saaspo, Aww)   │
└────────────────────────┘      └────────────────────────┘      └────────────────────────┘
```

## 1. Real-World User Flows & App UX
*Best for studying complete user journeys, onboarding sequences, checkout configurations, and real-world shipped screens from leading tech companies.*

*   **[Mobbin](https://mobbin.com)** *(Free Tier Available)*
    *   **Focus**: Mobile apps (iOS & Android) and Web application screens.
    *   **Why use it**: The gold standard for seeing actual shipped product screens. You can browse by flows (e.g., onboarding, upgrade, search filter) and filter by specific UI patterns.
*   **[Page Flows](https://pageflows.com)** *(Free Previews Available)*
    *   **Focus**: User flow screen recordings.
    *   **Why use it**: Provides screencast videos of users performing specific tasks (e.g., creating accounts, deleting files, purchasing) on top web and mobile platforms.
*   **[Pttrns](https://pttrns.com)** *(Free Previews Available)*
    *   **Focus**: Mobile interface design patterns.
    *   **Why use it**: A classic, highly structured library for browsing mobile screens categorized by task (e.g., notifications, search, profile, map views).

## 2. SaaS Dashboards & Interface Layouts
*Best for designing product internal interfaces, data grids, admin dashboards, configuration settings, and structural web app blocks.*

*   **[SaaSUI](https://saasui.design)** *(Free)*
    *   **Focus**: Shipped SaaS app dashboards and core sub-screens.
    *   **Why use it**: Curates actual shipped application screenshots categorized by view type (e.g., dashboards, empty states, profiles, settings, billing).
*   **[SaaS Interface](https://saasinterface.com)** *(Free)*
    *   **Focus**: Interactive software screen designs.
    *   **Why use it**: Ideal for comparison: compare how 10+ popular SaaS tools lay out their lists, filters, tables, and settings tabs.
*   **[Screenlane](https://screenlane.com)** *(Free)*
    *   **Focus**: Web & mobile dashboard UI elements.
    *   **Why use it**: Focuses heavily on isolated component categories (e.g. headers, sidebars, activity streams, chart widgets).

## 3. SaaS Landing Pages & Marketing Design
*Best for public-facing homepage layouts, pricing pages, conversion hero sections, and marketing content grids.*

*   **[SaaSFrame](https://saasframe.io)** *(Free)*
    *   **Focus**: SaaS landing pages, marketing emails, and component layouts.
    *   **Why use it**: Helps copywriters and designers study SaaS hero headers, landing sections, pricing grids, and transactional emails.
*   **[Saaspo](https://saaspo.com)** *(Free)*
    *   **Focus**: Premium curated SaaS landing page design.
    *   **Why use it**: Highly aesthetic, modern landing page inspirations. Great for clean typography and product demo mockups.
*   **[Land-book](https://land-book.com)** *(Free)*
    *   **Focus**: Web landing pages across all industries.
    *   **Why use it**: Curated showcase of creative landing pages, portfolios, and product landing pages. Features a simple tag filtering system.
*   **[Lapa Ninja](https://www.lapa.ninja)** *(Free)*
    *   **Focus**: Large, searchable repository of live landing pages.
    *   **Why use it**: Includes full-page screenshots, source code resource suggestions, and templates. Excellent filter and search mechanism.
*   **[One Page Love](https://onepagelove.com)** *(Free)*
    *   **Focus**: Single-page website inspiration.
    *   **Why use it**: Focuses strictly on websites designed as a single continuous scroll, which is great for fast landing pages and quick portfolios.

## 4. High-End Creative Web Design & Portfolios
*Best for cutting-edge visuals, scroll-driven micro-animations, premium layout styling, WebGL/3D layouts, and agency portfolios.*

*   **[Godly](https://godly.website)** *(Free)*
    *   **Focus**: Highly aesthetic, motion-heavy interactive websites.
    *   **Why use it**: Curates pages utilizing modern visual trends, web-animations (GSAP), custom cursors, and layout concepts.
*   **[Awwwards](https://awwwards.com)** *(Free)*
    *   **Focus**: Award-winning web design and digital agency creative sites.
    *   **Why use it**: Shows what is possible in web design. Websites are graded on design, usability, creativity, and performance.
*   **[SiteInspire](https://siteinspire.com)** *(Free)*
    *   **Focus**: Clean, modern web design and interactive digital platforms.
    *   **Why use it**: Focuses on professional, commercial, and editorial design. It avoids flashy, laggy animations, highlighting clean type layout instead.
*   **[CSS Design Awards](https://www.cssdesignawards.com)** *(Free)*
    *   **Focus**: Visual website awards registry.
    *   **Why use it**: Great for CSS inspiration, color theories, layout patterns, and responsive UI ideas.

## 5. General Design & Aesthetic Repositories
*Best for discovering general design styles, moodboarding, color palettes, visual branding, and concept UI sketches.*

*   **[Behance](https://www.behance.net)** *(Free)*
    *   **Focus**: Full creative portfolios, case studies, and branding guides.
    *   **Why use it**: Excellent for reading deep-dive UI/UX case studies where designers explain their full process from wireframe to mockups.
*   **[Dribbble](https://dribbble.com)** *(Free)*
    *   **Focus**: Visual snippets, animations, and micro-interactions.
    *   **Why use it**: Great for visual UI ideas, logo concepts, and color schemes.
    *   *Note: Many designs here are concept art and do not follow real-world UX or developer feasibility guidelines.*
*   **[Savee](https://savee.it)** *(Free)*
    *   **Focus**: Minimalist and creative design moodboard repository.
    *   **Why use it**: Clean layout focused entirely on raw visual aesthetics, photos, branding elements, and web interfaces.

## 6. Interactive UX Checklist Tools
*Best for auditing design systems, verifying user interface flows, and ensuring compliance with UX principles.*

*   **[Checklist Design](https://www.checklist.design)** *(Free)*
    *   **Focus**: Detailed lists of components and page flows requirements.
    *   **Why use it**: Tells you exactly what elements are required for page types (e.g., what belongs on a Contact page) or components (e.g. what states a Button needs).
*   **[Laws of UX](https://lawsofux.com)** *(Free)*
    *   **Focus**: Core cognitive psychology guidelines for user interface design.
    *   **Why use it**: Explains fundamental rules (e.g. Fitts's Law, Miller's Law) that dictate successful product layouts.
*   **[UX Myths](https://uxmyths.com)** *(Free)*
    *   **Focus**: Debunking common UX misconceptions.
    *   **Why use it**: Collects evidence and studies to counter design assumptions (e.g. "Users don't scroll").
